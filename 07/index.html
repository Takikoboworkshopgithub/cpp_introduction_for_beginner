第07講：STL(2) 高速なデータ管理

「検索」と「保持」の極意
set, map, queue, stack を使いこなせ

★ 配列の限界を超え、アルゴリズムを加速せよ ★

0. 本講のゴール

  • set：重複を自動で削除し、瞬時に要素の有無を判定する
  • map：自由な「キー」でデータを紐付け、巨大な辞書を作る
  • queue：先入れ先出しで、幅優先探索（BFS）の基礎を固める
  • stack：後入れ先出しで、深さ優先探索（DFS）や逆順処理を理解する

1. 重複を許さない魔法の箱：set

set は、数学の「集合」と同じです。どんなに同じ値を insert しても、内部では1つしか保持されません。また、要素は常に自動で昇順に並べられます。 

set<int> s;
s.insert(100);
s.insert(50);
s.insert(100); // 重複は自動で消える

cout << "要素数: " << s.size() << endl; // 2

// 特定の値があるか検索（爆速：O(log N)）
if (s.count(50)) {
    cout << "50は存在します" << endl;
}

2. キーで引ける最強の辞書：map

配列の添字（a[i]）を数字以外（文字列など）に拡張したものです。

map<string, int> inventory;
inventory["Apple"] = 3;
inventory["Banana"] = 10;

// 加算も自由自在
inventory["Apple"] += 5;

cout << "Appleの在庫: " << inventory["Apple"] << endl; // 8

💡 データ構造の裏側

map や set は内部で「赤黒木」というバランスの良い木構造を構築しています。これにより、100万件のデータから1つの値を探す際も、わずか20回程度の比較（log₂ 100万）で完了します。

3. queue（キュー）：先入れ先出し (FIFO)

レジの行列や、プリンタの印刷待ちと同じ仕組みです。「最初に来たものが、最初に処理される」のが鉄則です。

【ビジュアルイメージ：トンネル】

[出口] ← (先にいれたもの) ← (あとでいれたもの) ← [入口]

queue<int> q;
q.push(1); // [1]
q.push(2); // [1, 2]
q.push(3); // [1, 2, 3]

while(!q.empty()){
    cout << q.front() << " "; // 先頭を参照
    q.pop(); // 先頭を削除
}
// 出力結果: 1 2 3

実用例： 迷路の最短経路を解く「幅優先探索（BFS）」では、この queue を使って「今いる場所から1歩で行ける場所」を順番に管理します。

4. stack（スタック）：後入れ先出し (LIFO)

机に積み上げた本や、ブラウザの「戻る」ボタンと同じ仕組みです。「最後に置いたものが、真っ先に取り出される」のが特徴です。

【ビジュアルイメージ：筒（底がある）】

(あとでいれたもの) ← [出口 / 入口]
(先にいれたもの) 
stack<int> st;
st.push(10); // [10]
st.push(20); // [10, 20]
st.push(30); // [10, 20, 30]

while(!st.empty()){
    cout << st.top() << " "; // 一番上を参照
    st.pop(); // 一番上を削除
}
// 出力結果: 30 20 10

実用例： プログラムの関数呼び出しの管理や、カッコの対応関係のチェック、迷路を奥深く突き進む「深さ優先探索（DFS）」などで活躍します。

set や map は「賢い管理」、queue や stack は「正しい順序の管理」を担当します。 これらを組み合わせることで、ただのデータの羅列が「意味のある情報」へと変わります。特に queue を使った幅優先探索ができるようになると、解ける問題の世界がグッと広がります。
