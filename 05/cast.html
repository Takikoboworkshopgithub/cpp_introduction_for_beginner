0. キャストのゴール

  • 変数の型を一時的に変換する「キャスト」の書き方を覚える
  • char 型と int 型を相互に変換し、ASCIIコードを自由に扱う
  • 整数同士の割り算で小数（実数）の計算結果を正しく導く
  • double 型から int 型への変換による切り捨て処理をマスターする

1. キャスト（型変換）とは？

プログラミングにおいて、変数の種類（データ型）を一時的に別の種類へと変身させることを キャスト（型変換） と呼びます。
C++では、変身させたい変数や値の前に (変身後の型名) と書くことで強制的に型を変えられます。

① 安全な変換（データの拡大）

char ➔ int など、小さな箱（1バイト）から大きな箱（4バイト）へデータを移す変換。データが破損する心配はありません。

② 情報が消える変換（データの縮小）

double ➔ int など、小数点以下を強制的に切り捨てる変換。あえて端数を消し去りたい時に重宝します。

2. 文字（char）と数値（int）の相互キャスト

前項の「ASCIIコード」の正体を、このキャスト機能を使って覗き見してみましょう。 

#include <bits/stdc++.h>
using namespace std;

int main() {
    // 【文字 ➔ 数値】 'a' の前に (int) をつけると、ASCIIコードが出力される
    cout << (int)'a' << endl;  // 97 と出力される

    // 【数値 ➔ 文字】 97 の前に (char) をつけると、対応する文字が出現する
    cout << (char)97 << endl;  // 'a' と出力される
}

このように、コンピュータ内部にある「背番号（整数）」をそのまま数値として見たいなら (int) を、数値に対応した「見た目の文字」に戻したいなら (char) を被せるだけで簡単に変換が可能です。

3. 競プロ実戦でのキャスト（計算の罠を回避する）

■ 罠：整数同士の割り算

C++では、整数（int）同士の割り算の答えは必ず整数（int）になる仕様があります。端数はすべて切り捨てられます。
5 割る 2 の答えとして 2.5 が欲しいのに、そのまま計算すると 2 に化けて大バグの原因になります。

■ 解決策：計算する瞬間に double へキャストする

割り算を行う際に、どちらか片方だけでも (double) にキャストすると、計算結果も小数になってくれます。 

#include <bits/stdc++.h>
using namespace std;

int main() {
    int a = 5;
    int b = 2;

    // ダメな例：int同士なので小数点以下が消滅する
    cout << a / b << endl;         // 2 と出力（大バグ）

    // 正しい例：片方を double 型にキャストして計算させる
    cout << (double)a / b << endl; // 2.5 と出力（大成功！）
    
    // 【補足】実数から整数にして、あえて小数点以下をバッサリ切り捨てたい時
    double pi = 3.141592;
    cout << (int)pi << endl;       // 3 と出力
}
