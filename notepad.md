## 1. 基本コマンド（Ubuntu/Linux）
ディレクトリ（フォルダ）の操作や、プログラムのコンパイルに必要な最小限のコマンドです。

| コマンド | フルスペル | 説明 |
| :--- | :--- | :--- |
| **`ls`** | **l**i**s**t | いまいる場所にあるファイルの一覧を表示 |
| **`mkdir [名]`** | **m**a**k**e **dir**ectory | 新しいフォルダ（ディレクトリ）を作成 |
| **`cd [名]`** | **c**hange **d**irectory | フォルダの中に移動（`cd ..` で1つ上に戻る） |
| **`cp [元] [先]`** | **c**o**p**y | ファイルをコピーする |
| **`mv [元] [先]`** | **m**o**v**e | ファイルを移動、または名前を変更する |
| **`g++ [名]`** | **G**NU **C++** | C++のコードを機械語に変換（コンパイル）する |
| **`./a.out`** | (実行) | コンパイルしてできた実行ファイルを動かす |

## 2. 開発が爆速になるTips
初心者が一番最初に覚えるべき、ストレスを激減させるテクニックです。

### ⌨️ Tabキー（補完）
長いフォルダ名やファイル名を一文字ずつ打つ必要はありません。
- 例：`cd cpp4...` まで打って **Tabキー** を押すと、`cd cpp4b/` と補完されます。

### ⬆️ キー（履歴）
一回打ったコマンドは履歴に残ります。
- **上矢印キー** を押すと、さっき打った `g++ main.cpp` などを一瞬で呼び出せます。

### ⚡ コマンドの連結 (&&)
コンパイルと実行を別々に打つのは面倒です。
- `g++ main.cpp && ./a.out` と打つと、**「コンパイルに成功したときだけ実行する」** という一連の流れが一度に終わります。

### 🛑 強制終了 (Ctrl + C)
もしプログラムが「無限ループ」に陥って止まらなくなったら、**Ctrlキーを押しながら C** を押してください。プログラムを即座に殺せます。

2. `cd practice` で中に入る。

### 第01講：C++への招待と計算の基本
- **内容:** `main`関数の書き方、`cin` / `cout`、変数、型（`int`, `double`, `long long`）。
- **周辺知識・豆知識:**
    - **型の限界:** Python勢がハマる「整数同士の割り算（`5 / 2 = 2`）」と、`int` の限界（オーバーフロー）を実験。

### 第02講：条件分岐と「論理」
- **内容:** `if`, `else if`, `else`、比較演算子、論理演算子（`&&`, `||`, `!`）。
- **周辺知識・豆知識:**
    - **0と1の論理:** `if (a % 2 != 0)` と `if (a % 2)` が同じ結果になる理由（C++における真偽値の扱い）。
    - **おまじないの正体:** `using namespace std;` や、爆速入出力 `cin.tie(0)` の紹介。
- **演習:** 閏年判定、FizzBuzz（if文版）。

### 第03講：繰り返し処理と「全探索」の入り口
- **内容:** `for`, `while`, `break`, `continue`、複合代入演算子。
- **周辺知識・豆知識:**
    - **マクロの導入:** `#define rep(i, n)` を使って、タイピング量を減らしミスを防ぐ「競プロ特有の文化」に触れる。
    - **計算量の感覚:** `time` コマンドを使い、2重ループで1億回回すとどのくらい時間がかかるか計測。
- **演習:** 1から$N$までの和、九九の表作成。

### 第04講：関数と「部品化」
- **内容:** 関数の定義、引数、戻り値、参照渡し (`&`)。
- **周辺知識・豆知識:**
    - **ポインタ不要の参照:** `swap(a, b)` を例に、ポインタを使わずに値を書き換える `&`（参照）の便利さを学ぶ。
    - **再帰の入り口:** 関数が自分自身を呼ぶ「再帰」の仕組みを視覚化。
- **演習:** 最大公約数（GCD）を求める関数作成。

## 中盤：データ構造とSTL編（第05講〜第07講）
C++の真骨頂であるSTLを使いこなし、実装スピードを上げる。

### 第05講：配列（vector）と文字列
- **内容:** `vector<int>`、`string`、多次元配列、拡張for文。
- **周辺知識・豆知識:**
    - **開発の自動化 (Bash):** `.bashrc` にエイリアスを作成。`g++` と `./a.out` を1コマンドに短縮し、作業効率を劇的に上げる。
    - **リダイレクト:** `./a.out < in.txt` で、サンプル入力をファイルから自動で流し込む技。
- **演習:** 配列の合計、文字列の反転、成績処理。

### 第06講：STL(1) 便利ツールと並び替え
- **内容:** `sort`, `reverse`, `min`/`max`, `pair`, `abs`, `size`。
- **周辺知識・豆知識:**
    - **便利な短縮形:** `chmax(a, b)` マクロや、`pair` を使った複数データの同時ソート術。
    - **イテレータの初歩:** `v.begin(), v.end()` の意味を、「配列の端っこを指す矢印」として理解。
- **演習:** 座標（x, y）をxの昇順にソートする。

### 第07講：STL(2) 高速なデータ管理
- **内容:** `set`, `map`, `queue`, `stack`。
- **周辺知識・豆知識:**
    - **データ構造の裏側:** `map` はなぜ検索が速いのか？（バケットや木の概念）。
    - **浮動小数の闇:** `0.1 * 10` が `1.0` にならない誤差の罠と、`precision` の設定方法。
- **演習:** 重複を除いた要素数のカウント、辞書作成。
