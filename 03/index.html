第03講：繰り返し処理と「全探索」の設計

概念理解 → 設計判断 → 実装テンプレ → 実戦適用
全探索の原型を叩き込む重要回。

0. 本講のゴール

  • ループを「動き」で理解する

1. ループとは何か（実行順序の理解）

「条件を満たす限り、同じ処理に戻る」

2. while文：状態で止まるループ

  • 条件が「状態」で決まる
  • 回数は事前に分からない
#include <bits/stdc++.h>
using namespace std;

int main() {
    int hp = 10;
    int count = 0;

    while (hp > 0) {
        cout << "攻撃！ HP=" << hp << endl;
        hp -= 3;
        count++;
    }

    cout << "終了: hp=" << hp << ", 回数=" << count << endl;
}

3. トレース（状態遷移の可視化）

「なんとなく動く」を禁止し、必ず追う

回数 hp(開始) hp -= 3 hp(終了) count
1 10 7 7 1
2 7 4 4 2
3 4 1 1 3
4 1 -2 -2 4

→ 4回回る（直感ミスに注意！）

4. for文：回数が決まるループ 

#include <bits/stdc++.h>
using namespace std;

int main() {
    int n = 5;

    for (int i = 0; i < n; i++) {
        cout << i << endl;
    }
}

構造

  • 初期化
  • 条件
  • 実行
  • 更新

5. for と while の関係 

// for
for (int i = 0; i < 5; i++) {
    cout << i << endl;
}

// while
int i = 0;
while (i < 5) {
    cout << i << endl;
    i++;
}

→ for は while の省略形

6. 使い分けの判断基準（重要）

構文 用途 判断基準
for 全探索 回数が決まっている
while シミュレーション 終了条件が状態

■ 実戦ルール

  • 書けるなら for を優先（安全）
  • while は「必ず止まる」と言い切れるときだけ使う

7. スコープ（変数の寿命） 

for (int i = 0; i < 3; i++) {
    cout << i << endl;
}
// cout << i << endl; ← iはスコープ外なのでエラー

int j;
for (j = 0; j < 3; j++) {}
cout << j << endl; // 事前に宣言したjはOK
■ 本質
{} は変数の「生存範囲」。狭いほど安全（バグが減る）。

8. break / continue（構造の制御）

■ break（打ち切り）

for (int i = 0; i < 10; i++) {
    if (i == 5) break;
    cout << i << endl;
}

→ 探索の終了（剪定）

■ continue（スキップ）

for (int i = 0; i < 10; i++) {
    if (i % 2 != 0) continue;
    cout << i << endl;
}

→ 不要なケースを除外

※ 注意：構造を壊すので多用は危険

9. 全探索とは何か（核心）

「すべての可能性を試す」

■ 基本テンプレ

for (int i = 0; i < n; i++) {
    if (/* 条件 */) {
        // 更新
    }
}

■ 二重ループ（組み合わせ）

for (int i = 0; i < n; i++) {
    for (int j = 0; j < m; j++) {
        // (i, j) を調べる
    }
}

→ 状態空間をなめる

10. 実戦例（全部入り） 

for (int i = 1; i <= 9; i++) {
    for (int j = 1; j <= 9; j++) {

        int val = i * j;

        if (val % 3 != 0) continue; // 3の倍数以外はスキップ

        if (val > 50) break; // 50を超えたら打ち切り

        cout << i << "x" << j << "=" << val << endl;
    }
}

11. 計算量の直感

■ 目安

  • 10^7 → 余裕
  • 10^8 → 1秒程度
  • 10^9 → 厳しい (TLE)

■ 例

  • O(N) → OK
  • O(N^2) → 危険

■ 実験コード

long long cnt = 0;

for (int i = 0; i < 10000; i++) {
    for (int j = 0; j < 10000; j++) {
        cnt++; // 10000 * 10000 = 10^8 (約1秒)
    }
}

cout << cnt << endl;

12. 典型的なバグ 

for (int i = 0; i <= n; i++) 
// ↑ ミス (off-by-oneエラー: = が不要な場合が多い)
while (hp > 0) {
    hp -= 3;
}
// ↑ 終了後の hp が 0 ぴったりになるとは限らない (負になる可能性)
