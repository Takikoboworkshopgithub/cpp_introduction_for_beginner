ICPC特有の入力形式と「考察」「実装」の考え方

C言語の基本文法や文字列操作を学んだら、
次はICPCの実戦的な形式と問題の解き方に進みましょう。

■ ICPC特有の入力形式

授業などで作成するプログラムでは、int n; scanf("%d", &n); のように一度だけ入力を受け取ることが多いでしょう。
しかしICPCでは、複数のテストケースを連続して処理する問題が頻繁に出題されます。

📄 例題：AOJ 1681 - 2025

与えられた整数 n に対して n × n の掛け算表の項目の総和を求める問題です。

Sample Input 1 (入力) 

9
1
2
100
0

Sample Output 1 (出力) 

2025
1
9
25502500

① 番兵値（sentinel）とは

この入力例は、以下のルールに従っています。

  • n = 9 なら 2025 を出力
  • n = 1 なら 1 を出力
  • n = 2 なら 9 を出力
  • n = 100 なら 25502500 を出力
  • n = 0 が来たら終了

ここで重要なのは、最後の 0 も入力データの一部であることです。
この 0 は「これ以上テストケースはありません」という合図として使われています。このような値を番兵値（sentinel）と呼びます。

② while(1) と break による実装

このような問題では、無限ループを作る while (1) と、ループを抜ける break を組み合わせて処理することが多くあります。入力を最後まで受け取る感覚を掴みましょう。 

#include <stdio.h>

typedef long long ll;

int main() {
    ll N;

    while (1) {
        scanf("%lld", &N);

        // 0 が来たらループを抜けて終了する
        if (N == 0) break;

        ll a = (N * (N + 1)) / 2;
        a *= a;

        printf("%lld\n", a);
    }

    return 0;
}

ICPCでは、0 が来たら終了、-1 が来たら終了、0 0 が来たら終了 のような入力形式が非常によく登場します。そのため、このループ処理のパターンには慣れておきましょう。

入力の受け取り方に慣れたら、次は問題の見方です。
ICPCでは、まず何を考えるかを整理し、そのあとでコードに落とし込むのが基本になります。

■ ICPCでは、問題を2段階で考える

競技プログラミングでは、「何をすればよいか考える（考察）」と、「それをプログラムにする（実装）」という2段階で問題を解きます。
実装で行き詰まったと感じるとき、実際にはコードではなく「考察の言語化」で止まっていることが多いため、この2つのステップを分けて見る練習をしてみましょう。

🔍 問題を読んでみよう：接頭辞と接尾辞を同じに

元の文字列 s を接頭辞にも接尾辞にも含む、s自身以外の最短の文字列を求める問題です。

2-1. 問題文を言い換える

まず、元の問題文を整理するために用語の定義を一度確認し、曖昧さをなくします。

例えば文字列 "test" を考えます。

  • 接頭辞（prefix）：先頭から連続する部分文字列（t, te, tes, test）
  • 接尾辞（suffix）：末尾まで連続する部分文字列（test, est, st, t）

この問題の目的は、「sを接頭辞にも接尾辞にも持つ、s自身以外の最短の文字列」を作ることです。例えば "test" なら答えは "testest" になります。

2-2. 考察（どういう性質に注目するか）

ここではまだコードのことは考えず、「考える部分」を独立させます。

① まず単純に並べてみる
文字列 "test" を2個並べると "testtest" になります。しかしこれは少し長すぎます。途中を重ねられないでしょうか。

② 重ねられる部分を探す
次のように並びをずらして観察してみます。 

test
   test

見てみると、先頭と末尾で t が重なっています。そこで "testest" という文字列が作れます。これは条件を満たしつつ最短になります。

③ 別の例を見る
"ttt" の場合、大きく重ねて "tttt" にできます。 

ttt
 ttt

④ 共通部分を探そう
実は、「文字列の接頭辞」と「文字列の接尾辞」のうち、もっとも長く一致する部分（最長一致部分）を探せばよいことが分かります。
例えば "ttt" なら "tt" が一致し、"cccpppcc" なら "cc" が一致しています。

2-3. 実装（コードに落とし込む）

方針が決まったので、ループや文字列操作を使ってコードを形にします。文字列長は最大50文字と非常に小さいため、すべての長さを順番に試しても十分間に合います。

■ 手順
長さ n - 1 から 1 まで逆順に試します。
"test" なら、"tes" と "est" ➔ "te" と "st" ➔ "t" と "t" の順に比較し、最初に一致したものを最長の共通部分（overlap）とします。
答えは、「元の文字列 ＋ 重なっていない後半部分」をくっつければ完成です（"test" + "est" = "testest"）。

C言語実装 

#include <stdio.h>
#include <string.h>

int main(void) {
    int n;

    while (scanf("%d", &n) == 1 && n != 0) {
        char s[55];
        scanf("%s", s);

        int overlap = 0;

        // 長い候補から順に一致するか確認
        for (int len = n - 1; len >= 1; len--) {
            // s と (s + n - len) から始まる len 文字を比較
            if (strncmp(s, s + n - len, len) == 0) {
                overlap = len;
                break;
            }
        }

        printf("%s%s\n", s, s + overlap);
    }
    return 0;
}

C++実装 

#include <iostream>
#include <string>

using namespace std;

int main() {
    int n;

    while (cin >> n && n != 0) {
        string s;
        cin >> s;

        int overlap = 0;

        for (int len = n - 1; len >= 1; len--) {
            bool ok = true;

            for (int i = 0; i < len; i++) {
                if (s[i] != s[n - len + i]) {
                    ok = false;
                    break;
                }
            }

            if (ok) {
                overlap = len;
                break;
            }
        }

        cout << s << s.substr(overlap) << '\n';
    }
    return 0;
}

■ この問題で学んでほしいこと

この問題の本質は、「どの関数を使うか」ではなく、「何を調べれば答えになるのか（接頭辞と接尾辞の最長一致部分を探す）」を見つけることです。
ICPCでは、このように【 考察 ➔ 実装 】という順番を意識するだけで、コードを書き始めて迷子になるリスクを大きく減らすことができます。まず紙の上で解法を整理し、そのあとでコードに落とし込む習慣を身につけていきましょう。
