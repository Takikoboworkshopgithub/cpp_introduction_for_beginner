1. for文の中で変数宣言できる (C17)

大学の授業（C89ベース）では、関数の最初に変数をまとめて宣言していると思います。
しかしICPCで利用できるC17環境では、コードの途中での変数宣言が可能です。必要だと思ったらその時に記述できるので書きやすい。また、for文のカッコ内で直接変数宣言が可能です。

授業の書き方 (C89) 

#include <stdio.h>

int main(void)
{
    int i; /* 上で宣言しておく */

    for(i = 0; i < 5; i++){
        printf("%d\n", i);
    }
    return 0;
}

ICPCの書き方 (C17) 

#include <stdio.h>

int main(void)
{
    /* for文の中で直接宣言できる！ */
    for(int i = 0; i < 5; i++){
        printf("%d\n", i);
    }
    return 0;
}



💡 なぜ便利？
変数 i の有効範囲を for 文の中だけに限定でき、バグが減ります。競技プログラミングではこちらの書き方が一般的です。

2. 可変長配列(VLA) と グローバル配列

■ 実行時に入力サイズに合わせて配列を作る

授業では int a[1000]; のように固定サイズで配列を作っていましたが、競プロでは「N個の整数が与えられる」のように、入力サイズが実行するまでわかりません。
C17では、入力された変数Nを使って動的に配列を作ることができます。 

#include <stdio.h>

int main(void)
{
    int n;
    scanf("%d", &n);

    int a[n]; // 入力されたNのサイズで配列を作る！

    for(int i = 0; i < n; i++){
        scanf("%d", &a[i]);
    }
    return 0;
}

⚠️ 巨大すぎる可変長配列の罠

int a[100000000]; のような巨大な配列を関数の内側（mainの中など）で作ると、スタック領域の容量不足により Segmentation Fault（セグフォ） で強制終了する可能性があります。

■ 巨大な配列は「グローバル配列」で解決

競技プログラミングでは、巨大なサイズの配列を確保する場合、main 関数の外側（グローバル領域）に配置するのが定石です。スタック領域とは別の場所に確保されるため、安全に巨大な配列を作れます。 

#include <stdio.h>

#define MAXN 100000

/* 関数の外に置くことで巨大な配列も安全に確保 */
int a[MAXN];

int main(void)
{
    int n;
    scanf("%d", &n);

    for(int i = 0; i < n; i++){
        scanf("%d", &a[i]);
    }
    return 0;
}

3. 提出が通らない！？「よくある３大間違い」

ICPCの練習や本番で、特に1年生が陥りやすいエラーの原因をまとめました。

❌ 罠1：配列の範囲外アクセス

配列の要素数は N 個確保した場合、インデックスは 0 から N-1 までしか使えません。a[N] にアクセスすると予期せぬ動作をします。 

int a[5];
// 使えるのは a[0], a[1], a[2], a[3], a[4] のみ！
a[5] = 10; // ← エラー！確保した領域の外を触っている（範囲外アクセス）

❌ 罠2：scanf の & (アンパサンド) 忘れ

数値を入力させるときに & を忘れると、ほぼ確実に Segmentation Fault となり提出しても Runtime Error (RE) になります。 

int n;
scanf("%d", n);  // ← 誤り！ & が無いのでプログラムがクラッシュする
scanf("%d", &n); // ← 正解！ 変数のメモリ番地を渡す必要がある

❌ 罠3：文字列を入れる配列のサイズ不足

「5文字の文字列が入力される」問題だからといって、char s[5]; と宣言してはいけません。文字列の最後には必ず 終端文字 \0 が必要になるため、文字数＋1のサイズ（安全に少し大きめ）を確保する必要があります。 

char s[5];
scanf("%s", s); // もし "hello" と入力されると、\0 を入れるための6個目の箱がなくてバグる！

// 正しくは、余裕を持って大きめのサイズを宣言しておく
char s[100];
scanf("%s", s);
