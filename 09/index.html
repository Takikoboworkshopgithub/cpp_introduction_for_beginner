1. ICPCってなに？

ICPCは、3人1組のチームでコンピュータを囲み、制限時間内にどれだけ多くの問題を解けるかを競う世界規模 の歴史あるコンテストです。
どうやって実装して問題を解決するかや、計算量の正しい見積もりが大事なコンテストです！

順位付けのルール

基本的には「問題を早く、多く解いたチームの勝ち」です！具体的なタイブレークは以下の通りです。

  • 正解数（AC数）： 多いチームが当然上位。
  • ペナルティ（総経過時間）： 正解数が並んだ場合、各問題を正解するまでにかかった「経過時間の合計」に、正解するまでに犯した「誤答1回につき20分」を加算した値が少ないチームが上位になります。

3. コンテストを生き抜く「計算量」の知識

プログラムを実行したとき、時間切れ（TLE）にならないかを提出前に見積もるスキルは競プロ最重要項目です。
計算量とは、そのプログラムが何回計算するのかを見積もるもので、ICPCでは「1秒間に約10^8回の計算が可能」という経験則がよく使われます。

⏳ 1秒間に処理できる計算の目安： 約 10^8 回（1億回）

🚨 10^9 回（10億回）は基本的にアウト！ 2秒の制限時間があってもほぼ確実にTLEになります。

⚖️ 10^8 回前後の「定数倍」のセーフ・アウトの壁：

  • 【セーフ（定数倍が軽い）】 単純な配列の読み書き、forループ内での足し算・引き算、ビット演算。
  • 【アウト（定数倍が重い）】 ループ内での map や set の頻繁な操作、大量の割り算（%）、複雑な多次元配列へのアクセス。

セクション1：まずは「何回くり返すか」を見る
[ループ] Q1 / 14

1からNまで表示する

for (int i = 0; i < N; i++) {
    printf("%d\n", i);
}
for (int i = 0; i < N; i++) {
    cout << i << "\n";
}
for (int i = 0; i < N; i++) {
    System.out.println(i);
}
for i in range(N):
    print(i)

❌ O(N) は不正解

🟢 Correct!

1重のループなので、素直にN回処理されます。

→ 正解: O(N)

[ループ] Q2 / 14

二重ループ

for (int i = 0; i < N; i++) {
    for (int j = 0; j < N; j++) {
        printf("%d\n", i + j);
    }
}
for (int i = 0; i < N; i++) {
    for (int j = 0; j < N; j++) {
        cout << i + j << "\n";
    }
}
for (int i = 0; i < N; i++) {
    for (int j = 0; j < N; j++) {
        System.out.println(i + j);
    }
}
for i in range(N):
    for j in range(N):
        print(i + j)

❌ 不正解です

🟢 Correct!

外側だけでなく内側でもN回まわるため、N × N 回処理されます。

→ 正解: O(N^2)

for (int i = 0; i < N; i++) {

セクション2：「足し算」と「掛け算」
[ループ] Q4 / 14

ループが連続している場合

for (int i = 0; i < N; i++) {}
for (int i = 0; i < N; i++) {}
for (int i = 0; i < N; i++) {}
for (int i = 0; i < N; i++) {}
for (int i = 0; i < N; i++) {}
for (int i = 0; i < N; i++) {}
for i in range(N): pass
for i in range(N): pass

❌ 不正解です

🟢 Correct!

入れ子（二重）ではないので掛け算になりません。N + N = 2N ですが、オーダー記法では定数倍を無視するので O(N) です。

→ 正解: O(N)

[ループ] Q5 / 14

N と M が別の二重ループ

for (int i = 0; i < N; i++) {
    for (int j = 0; j < M; j++) {
        sum++;
    }
}
for (int i = 0; i < N; i++) {
    for (int j = 0; j < M; j++) {
        sum++;
    }
}
for (int i = 0; i < N; i++) {
    for (int j = 0; j < M; j++) {
        sum++;
    }
}
for i in range(N):
    for j in range(M):
        sum += 1

❌ 不正解です

🟢 Correct!

外側が N回、内側が M回 でしっかりと二重になっているため、掛け算の O(NM) になります。

→ 正解: O(NM)

セクション3：O(1) を知る
[探索] Q6 / 14

数式で一発計算

long long sum = (long long)N * (N + 1) / 2;
long long sum = (long long)N * (N + 1) / 2;
long sum = (long)N * (N + 1) / 2;
sum = N * (N + 1) // 2

❌ 不正解です

🟢 Correct!

ループが一切ありません。Nがどれだけ天文学的な数字になっても、計算回数は固定（1回）です。

→ 正解: O(1)

[探索] Q7 / 14

配列アクセス

printf("%d", A[500]);
cout << A[500];
System.out.println(A[500]);
print(A[500])

❌ 不正解です

🟢 Correct!

配列は「先頭から順番に探す」必要がありません。添え字（インデックス）を指定すれば、一発で直接取り出せます。

→ 正解: O(1)

セクション4：logN の導入
[探索] Q8 / 14

半分ずつ減る処理

while (N > 1) {
    N /= 2;
}
while (N > 1) {
    N /= 2;
}
while (N > 1) {
    N /= 2;
}
while N > 1:
    N //= 2

❌ 不正解です

🟢 Correct!

「半分ずつ減る」処理は一瞬で終わるため超高速になります（例: 16→8→4→2→1 のわずか4回）。この「半分ずつ減ると速い」動きを O(log N) と呼びます。

→ 正解: O(log N)

[STL] Q9 / 14

map の検索

// ※C言語には標準の連想配列(map)がありません
// 二分探索によるキー検索が行われているとイメージしてください
int value = map_search(mp, "apple");
map<string, int> mp;
cout << mp["apple"];
// ※計算量がO(log N)になるツリー構造のマップとします
TreeMap<String, Integer> mp = new TreeMap<>();
System.out.println(mp.get("apple"));
# ※Pythonの標準dict(ハッシュ)ではなく、
# 二分探索ツリー構造を使った連想配列の検索と仮定します
print(mp["apple"])

❌ 不正解です

🟢 Correct!

C++の `std::map` は、内部で探す範囲を少しずつ（半分ずつ）減らしながら検索をかける賢い仕組みを持っています。

→ 正解: O(log N)

[STL] Q10 / 14

set の検索

// ※C言語には標準の集合(set)がありません
// 指定の値が含まれているかのツリー検索をイメージしてください
bool has_x = set_contains(st, x);
set<int> st;
cout << st.count(x);
// ※計算量がO(log N)になるツリー構造のセットとします
TreeSet<Integer> st = new TreeSet<>();
System.out.println(st.contains(x));
# ※Pythonの標準集合ではなく、
# 二分探索ツリー構造を使った集合の検索と仮定します
print(x in st)

❌ 不正解です

🟢 Correct!

set も map と同様です。内部でデータが自動的に整理されて保持されているため、高速な探索（範囲を半分ずつに絞るような動き）が可能です。

→ 正解: O(log N)

セクション5：競プロでよく見る計算量
[STL] Q11 / 14

sort

// N個の配列Aを並び替える
qsort(A, N, sizeof(int), compare);
sort(A.begin(), A.end());
Arrays.sort(A);
A.sort()

❌ 不正解です

🟢 Correct!

大量のデータを最も高速に並び替える標準的なアルゴリズムの計算量です。競技プログラミングにおいて、一重ループの次に出番が多い超重要計算量です。

→ 正解: O(N log N)

[STL] Q12 / 14

set に N 個追加

// N個の要素を順にツリー構造のセットに追加していくとイメージしてください
for (int i = 0; i < N; i++) {
    set_insert(st, A[i]);
}
set<int> st;
for (int i = 0; i < N; i++) {
    st.insert(A[i]);
}
TreeSet<Integer> st = new TreeSet<>();
for (int i = 0; i < N; i++) {
    st.add(A[i]);
}
# ※二分探索ツリー構造のセットに要素をN回追加すると仮定します
for i in range(N):
    st.add(A[i])

❌ 不正解です

🟢 Correct!

1回の追加(insert/add)にかかる計算量は O(log N) です。それを外側のループで N回 繰り返しているので、掛け算になって O(N log N) になります。

→ 正解: O(N log N)

セクション6：引っかけ問題
[ループ] Q13 / 14

見た目は二重ループ

int j = 0;
for (int i = 0; i < N; i++) {
    while (j < N) {
        j++;
    }
}
int j = 0;
for (int i = 0; i < N; i++) {
    while (j < N) {
        j++;
    }
}
int j = 0;
for (int i = 0; i < N; i++) {
    while (j < N) {
        j++;
    }
}
j = 0
for i in range(N):
    while j < N:
        j += 1

❌ 不正解です

🟢 Correct!

変数 `j` が外側で初期化され、ループ内でリセットされていません。そのため、内側の `while` 文はプログラム全体を通して「合計で N 回」しか動きません。形に騙されないようにしましょう。

→ 正解: O(N)

セクション7：実戦感覚
[実戦] Q14 / 14

N = 100000 (10万) のとき、実行が厳しい（間に合わない）計算量は？

※この問題にプログラムコードはありません。選択肢を直接選んでください。

❌ 不正解です

🟢 Correct!

10万の2乗は「100億回」に近い処理になってしまいます。競技プログラミングの一般的な時間制限（2秒）では、1秒間に処理できる限界がだいたい1億〜数億回なので、絶対に間に合いません。

→ 正解: O(N^2)
