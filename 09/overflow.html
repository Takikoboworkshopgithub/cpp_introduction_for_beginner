int型の限界を超えると、なぜかマイナスの値になったりして、全く見当外れな結果になることがあります。
これが「オーバーフロー」と呼ばれる現象で、ICPCの計算問題では常に警戒すべき落とし穴です。

沈黙のバグ「オーバーフロー」

Pythonなどの言語は自動で桁数を拡張してくれますが、C++やJavaといった競技プログラミングの主流言語では、変数の型（箱の大きさ）が厳格に決まっています。
箱の許容量を超えて値が溢れ出す現象をオーバーフローと呼び、エラーも出ずに全く見当外れなマイナスの値などに化けてしまうため、看過できない重大なバグの原因となります。

📦 int型の限界： -2,147,483,648 〜 2,147,483,647 (約 2 × 109)

🚀 long long型 (Javaは long型) の限界： 約 ±9 × 1018

🚨 ICPCの計算問題では、答えだけでなく「計算の途中結果」がintの限界を突破しないか常に留意する必要があります。

[足し算] Q1 / 5

限界を超えた足し算

int x = 2000000000;
x += 1000000000;
printf("%d\n", x);
int x = 2000000000;
x += 1000000000;
cout << x << endl;
int x = 2000000000;
x += 1000000000;
System.out.println(x);

❌ 30億は int型 では保持できません。

🟢 Correct!

int型の最大値は約21億です。これを超過すると、内部のビット表現の都合上、突然マイナスの値（-1294967296 など）に反転してしまいます。

[掛け算] Q2 / 5

10万 × 10万

int a = 100000;
int b = 100000;
printf("%d\n", a * b);
int a = 100000;
int b = 100000;
cout << a * b << endl;
int a = 100000;
int b = 100000;
System.out.println(a * b);

❌ 100億は int型 の限界を超えています。

🟢 Correct!

10万という数値自体はint型に余裕で収まりますが、掛け算をした瞬間に100億となり、限界の21億を優に突破してしまいます。

[総和] Q3 / 5

ループの中の罠

int sum = 0;
for(int i = 0; i < 100000; i++){
    sum += 100000;
}
printf("%d\n", sum);
int sum = 0;
for(int i = 0; i < 100000; i++){
    sum += 100000;
}
cout << sum << endl;
int sum = 0;
for(int i = 0; i < 100000; i++){
    sum += 100000;
}
System.out.println(sum);

❌ 蓄積される値が 21億 を超える瞬間に破綻します。

🟢 Correct!

足し算であっても、蓄積されていく変数 `sum` の型が int である以上、合計が21億を超えた瞬間に崩壊します。

ここからが本番：競技プログラミング最大の陥穽
[キャスト] Q4 / 5

受け皿を大きくしたから安心？

int a = 100000;
int b = 100000;
long long c = a * b; // 大きな型に代入！
printf("%lld\n", c);
int a = 100000;
int b = 100000;
long long c = a * b; // 大きな型に代入！
cout << c << endl;
int a = 100000;
int b = 100000;
long c = a * b; // 大きな型に代入！
System.out.println(c);

❌ 残念ながら、この記述は顕著なバグを生み出します。

🟢 Correct!

一見正しいように見受けられますが、右辺の a * b は「int型同士の計算」として処理されるため、大きな箱(c)に入れる前に空中でオーバーフローしてしまいます。壊れた値が代入されるという、非常に発見しづらい陥穽です。

[キャスト] Q5 / 5

正しいオーバーフロー回避術

int a = 100000;
int b = 100000;
// 1LLを掛けるか、(long long)で明示的に型変換する
long long c = 1LL * a * b; 
printf("%lld\n", c);
int a = 100000;
int b = 100000;
// 1LLを掛けるか、(long long)で明示的に型変換する
long long c = 1LL * a * b; 
cout << c << endl;
int a = 100000;
int b = 100000;
// 1Lを掛けるか、(long)で明示的に型変換する
long c = 1L * a * b; 
System.out.println(c);

❌ 不正解です。この記述こそが正しい回避策です。

🟢 Correct!

計算式の先頭で 1LL (Javaなら 1L) を掛けるか、(long long)a とキャストすることで、右辺の計算式全体が「巨大な箱での計算」に昇格し、無事に正しい結果を得ることができます。
