第04講：関数と「部品化」の設計

概念理解 → 設計判断 → 実装テンプレ → 実戦適用
重複を削り、プログラムを「構造化」する重要回。

★ 同じことを何回も書くな ★

1. 問題：「これ、何回書く？」 

cout << 3 + 5 << endl;
cout << 10 + 20 << endl;
cout << 7 + 8 << endl;
cout << 100 + 200 << endl;

👉 同じ形を4回書いている

もしこれが100回あったら？

違和感：
・コピペが増える
・修正が面倒になる
・バグが混ざる

→ まとめて1つにできないか？

int calc(int x, int y){
    return x + y;
}

int main(){
    cout << calc(3,5) << endl;
    cout << calc(10,20) << endl;
    cout << calc(7,8) << endl;
    cout << calc(100,200) << endl;
}
関数 = 「同じ処理を1箇所にまとめる」

2. return（最重要） 

int add(int x, int y){
    return x + y;
}
return を書かないと「何も返らない」

※ int は「何を返すか」の型

int result = add(3,5);
cout << result << endl;

👉 値として使えるのが重要

3. void型

void calc(int x, int y){
    cout << x + y << endl;
}

int main(){
    calc(3,5);
    calc(10,20);
    calc(7,8);
    calc(100,200);
}

👉 「結果を使わないとき」

使い方 選択
結果を計算して使う int
表示だけする void
⚠ printだけの関数は再利用しにくい

4. 値渡し vs 参照渡し

■ 値渡し（変わらない）

void add100(int x){
    x += 100;
}

int main(){
    int a = 10;
    add100(a);
    cout << a << endl; // 10
}

■ 参照渡し（変わる）

void add100(int &x){
    x += 100;
}

int main(){
    int a = 10;
    add100(a);
    cout << a << endl; // 110
}
& = 同じデータを共有している

5. swapの正体 

int main(){
    int a = 3, b = 5;

    int tmp = a;
    a = b;
    b = tmp;

    cout << a << " " << b << endl;
}
行 a b tmp
初期 3 5 -
tmp=a 3 5 3
a=b 5 5 3
b=tmp 5 3 3

👉 入れ替わった

void my_swap(int &a, int &b){
    int tmp = a;
    a = b;
    b = tmp;
}

👉 & がないと外では変わらない

6. 再帰とは何か

関数が自分自身を呼び出す

■ イメージ

count_down(3)
 ↓
3 を表示
count_down(2)
 ↓
2 を表示
count_down(1)
 ↓
1 を表示
count_down(0)
 ↓
終了 
void count_down(int n){
    if(n == 0) return;
    cout << n << endl;
    count_down(n-1);
}

👉 「問題を小さくしている」

問題3：値の更新

整数 a を受け取り、その値を +50 する関数を作れ。呼び出し元の値も変化するようにせよ。

👉 & は必要か？

問題4：設計判断

2つの整数のうち大きい方を求める関数を作れ。

👉 「値を返す」のか「書き換える」のか？

📚 まとめ

  • 関数 = 重複を消す
  • return = 値を返す
  • void = 出力専用
  • & = データ共有

🚀 設計指針

「返すか？書き換えるか？」を必ず判断する。基本は return で値を返し、呼び出し元で代入するのが安全で美しい設計。
