第08講：モダン文法と構造体

C++をもっと楽に、もっとスマートに書く技術。
バラバラのデータを「構造体」で制圧せよ！

★ 基礎編の総仕上げ！バグのないコードを目指せ ★

1. 型推論の魔法：auto

これまで int や string など、変数には必ず「型」を指定してきました。しかし、モダンなC++には右辺の値から型を自動で推測してくれる auto という魔法のキーワードがあります。 

#include <bits/stdc++.h>
using namespace std;

int main() {
    auto a = 10;       // 勝手に int だと判断してくれる！
    auto b = 3.14;     // 勝手に double だと判断！
    auto c = "hello";  // 文字列もOK

    cout << a << " " << b << endl;
}

※注意： 便利ですが、最初は「後で紹介するある技術」と組み合わせる時だけ使うのがおすすめです。その威力を次で見てみましょう！

2. 拡張for文（スマートなループ）

配列の中身をすべて出力したい時、今まで私たちはこう書いていました。

vector<int> A = {10, 20, 30, 40};

// 今までの書き方（iの範囲を間違えやすい…）
for (int i = 0; i < A.size(); i++) {
    cout << A[i] << " ";
}

これを圧倒的に見やすく、書きやすくしたのが「拡張for文」です。

vector<int> A = {10, 20, 30, 40};

// 拡張for文！ Aの中身を順番に x に取り出してループする
for (int x : A) {
    cout << x << " ";
}

さらに、ここで先ほどの auto が合体すると最強の書き方になります。 

// 型が何であれ、勝手に推論して回してくれる！
for (auto x : A) {
    cout << x << " ";
}

これなら first や second の打ち間違えでバグらせる心配も一切ありません。競技プログラミングでも必須の超便利テクニックです！

💡 なぜ拡張for文で auto を使うの？

「int くらい自分で書くよ」と思うかもしれません。では、第06講でやった pair型（座標のリストなど） をループで回す場合を比較してみましょう。 

// 座標のリスト {x, y}
vector<pair<int, int>> points = {{1, 2}, {3, 4}, {5, 6}};

// 【auto なし】型名が長すぎて打ち間違えるし、読むのもウッとなる…
for (pair<int, int> p : points) {
    cout << p.first << " " << p.second << endl;
}

// 【auto あり】どんなに複雑な型でも、たった4文字でスマートに解決！
for (auto p : points) {
    cout << p.first << " " << p.second << endl;
}

このように、拡張for文と auto のコンボは、複雑なデータ構造になればなるほど圧倒的な威力を発揮します！

🚀 さらにモダンに！「構造化束縛」（C++17）

先ほどの pair 型のループ、p.first や p.second と書くのは少し面倒ですし、どっちがどっちか分かりにくかったですよね。
モダンなC++では、なんと中身をその場でバラして別々の変数に一発で代入する強力な書き方があります！ 

#include <bits/stdc++.h>
using namespace std;

int main() {
    // 座標のリスト {x, y}
    vector<pair<int, int>> points = {{1, 2}, {3, 4}, {5, 6}};

    // 拡張for文の中で [x, y] のように書くと、勝手に分解してくれる！
    for (auto [x, y] : points) {
        cout << "X座標: " << x << ", Y座標: " << y << endl;
    }
}

これなら first や second の打ち間違えでバグらせる心配も一切ありません。
コードが圧倒的に直感的で読みやすくなるため、競技プログラミングでも必須の超便利テクニックです！

3. データを直感的に束ねる：構造体（struct）

生徒の「国語」「英語」「数学」の点数をまとめて管理するプログラムを想像してください。 これまで通りに変数を別々に作ると、管理が非常に面倒になります。 

// 変数を別々に作ると、誰のデータか分からなくなりがち…
int japanese;
int english;
int math;

これを美しく解決するのが「構造体（struct）」です。複数の異なるデータを1つにまとめた「新しいデータ型」を自分で作ることができます。 

#include <bits/stdc++.h>
using namespace std;

// 成績をまとめた「Score」という新しい型を自分で作る！
struct Score {
    int japanese;
    int english;
    int math;
}; // ← 最後にセミコロン「;」が必要な点に注意！

int main() {
    // Score型の変数「p」を作る
    Score p;

    // ドット「.」を使って、中身に直感的にアクセスできる！
    p.japanese = 80;
    p.english = 90;
    p.math = 100;

    cout << "数学の点数: " << p.math << "点" << endl;
}

1人分のまとめ方は分かりました。では、**「生徒10人分」**のデータを管理したい時はどうすればいいでしょうか？ さきほど作った `Score` 型を、そのまま vector の配列にしてしまえば解決します！ 

    // Score型が入る、サイズ10の配列「students」を作る
    vector<Score> students(10);

    // 出席番号5番（インデックス4）の生徒の成績を入力
    students[4].japanese = 80;
    students[4].english = 90;
    students[4].math = 100;

    // 全員の数学の点数を出すなら、先ほどの auto × 拡張for文が大活躍！
    for (auto s : students) {
        cout <≪ s.math << " ";
    }

もし、これと同じ「10人分の3教科のデータ」を、構造体を使わずに1つの配列で管理しようとすると、以下のような「2次元配列」を使うしかありませんでした。 

    /* 3つの大きな箱(国/英/数)の中に サイズ10の配列 があるイメージ */
    vector<vector<int>> test(3, vector<int>(10));
    
    /* 出席番号5番（インデックス4）の数学（インデックス2）の成績を表示 */
    cout << test[2][4] << endl;

比較してみましょう！

❌ 2次元配列： test[2][4] （2って何番だっけ？と頭が混乱する）
⭕ 構造体の配列： students[4].math （4番の生徒の、math！と一発でわかる）

このように、配列の中に構造体を載せることで、複雑なデータもバグを埋め込むことなく圧倒的に直感的に処理できるようになります。

5. おまけ：switch文

最後に、条件分岐のもう一つの書き方を紹介します。特定の値（1なら〇〇、2なら〇〇）によって処理を分けたい時、if 文を並べるよりスッキリ書ける switch 文です。 

#include <bits/stdc++.h>
using namespace std;

int main() {
    int command;
    cin >> command;

    switch (command) {
        case 1:
            cout << "攻撃！" << endl;
            break; // ← breakを忘れると次の処理も実行されてしまうので注意！
        case 2:
            cout << "防御！" << endl;
            break;
        default:
            // どの case にも当てはまらない場合
            cout << "逃げる" << endl;
            break;
    }
}
