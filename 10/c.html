C問題：水たまりの容量

この問題は、水槽全体の形を見て解こうとすると難しく感じます。しかし、調べたい区画を1つだけ選び、「この区画には水が残るだろうか？」という視点に切り替えると、解法が自然と見えてきます。競技プログラミングでは、このように「全体」ではなく「1要素」に注目して法則を見つける考え方がよく登場します。

★ 視点を極限までミクロに変えてみよう ★

1. 問題の解釈

まず問題を簡単な日本語に言い換えてみます。

水槽の底は凸凹しています。満水だった水槽の両端が開いたので、水は低い場所から流れ出ます。最終的に各区画にどれだけ水が残るかを求め、その合計を計算する問題です。

一見すると水槽全体を考えたくなりますが、それでは複雑になります。そこで、「1つの区画（マス）だけ」に注目してみます。

各区画の残る水 ➔ 全部足す ➔ 答え という方針で解き進めましょう。

2. 水が残る条件

ある区画に水が残るためには、「左側に壁がある」かつ「右側にも壁がある」の両方が必要です。どちらか片方しか壁がない場合、水はその方向へ流れ出てしまいます。

【図①：水が残るパターン】

この区画の左右にはどちらも高い場所があります。そのため、この区画には水が残る可能性があります。

【図②：片方に壁がないパターン】

左側に高い壁があっても、右側が低ければ水は右へ流れてしまいます。同様に、右側だけ壁があっても左へ流れてしまいます。

3. 水面はどう決まる？

左右に壁があれば水は残りますが、水面は好きな高さまで上げられるわけではありません。

【図③：水面は低い壁で決まる】

例えば、左の壁が高さ8、右の壁が高さ5、真ん中が高さ2だとします。
ここに高さ6まで水を入れると、右側（高さ5）の壁を越えて溢れ出してしまいます。そのため、水面は左右の低い方の壁までしか上がることができません。

水面 = min(左の壁, 右の壁)

4. 左右の壁の検索と、max / min の役割

では、「左の壁」とは何でしょうか。実は、「その区画より左側にある壁の中で、一番高い壁」のことです。右側も同様です。

実際のコードでは、以下のようにループを回して求めています。

int l = 0, r = 0;
for (int j = 0; j < i; j++)   l = max(l, d[j]); // l：左側で一番高い壁
for (int j = i + 1; j < n; j++) r = max(r, d[j]); // r：右側で一番高い壁

💡 なぜ max と min の両方があるの？

ここは初心者が一番混乱しやすい部分です。このコードでは d[i] = 1000 - d[i]; として、深さではなく高さに変換しています。そのため、一番高い壁を探すときは max() を使います。

その後、見つかった左右の壁の低い方を水面にするために min(l, r) を使います。二つの関数には全く別の役割があります。

処理 意味・役割
max() 左右でそれぞれ一番高い壁を探す
min() 左右の壁を戦わせ、低い方の壁を水面にする
max(..., 0) 計算結果が負になったら0にする（水は残らない）

5. 残る水の計算

水面が決まれば、その区画に残る水は 「水面 − 区画の高さ」 です。

【図④：残る水の量】

コードでは ans += max((min(l, r) - d[i]), 0); の1行がこれに対応しています。
ここで最後に max(..., 0) を使っているのは、自分が壁そのもので値が負になってしまう場合に、溜まる水を 0 にするためです。これをすべての区画について全部足し合わせます。

6. 実装コード

考察の流れが、そのまま上から順にコード化されていることを確認してください。 

#include<bits/stdc++.h>
using namespace std;

bool solve(){
    int n; cin >> n;
    if (n == 0) return false;

    vector<int> d(n);
    for (int i = 0; i < n; i++) {
        cin >> d[i];
        d[i] = 1000 - d[i]; // 深さではなく高さに変換
    }

    int ans = 0;
    for (int i = 0; i < n; i++){
        int l = 0, r = 0;
        // 左側で一番高い壁を探す (max)
        for (int j = 0; j < i; j++) l = max(l, d[j]);
        // 右側で一番高い壁を探す (max)
        for (int j = i + 1; j < n; j++) r = max(r, d[j]);

        // 水面を決めて(min)、残る水を足す。負なら0。
        ans += max((min(l, r) - d[i]), 0);
    }
    cout << ans << endl;
    return true;
}

int main(){
    while(solve());
}

7. 計算量

各区画について、毎回独立して左側と右側をすべて見に行きます。
そのため、全体の計算量は O(N²) となります。

問題の制約は N ≤ 100 なので、計算回数は最大でも 10,000 回程度。実行時間制限に対して十分高速に、余裕を持って間に合います！
