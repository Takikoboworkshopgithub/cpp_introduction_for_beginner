B問題：自動販売機

一方向への走査と、その場での最善択の選択
競技プログラミングの超重要手法『貪欲法（Greedy）』の基礎を掴む！

★ ギリギリまで欲張って、最小の設置数を目指せ ★

1. 問題文（要約）＆ 解釈・翻訳

廊下にあるすべての扉について、自販機までの距離が d 以下 になるように自販機を設置したいです。必要な自販機の最小台数を求める問題を考えてみましょう。

自販機は廊下の好きな場所に置くことができ、設置した場所から左に d、右に d の範囲にある扉をカバーできます。

自販機 (●)

＜－－－－ d －－－－＞★＜－－－－ d －－－－＞

つまり、この問題は表裏を返すと「長さ 2d の区間で、すべての扉を覆う問題」と言い換えることができます。

ここで重要な境界条件は、「① 扉は左から順番に並んでいる」「② 自販機は好きな場所に置ける」の2点です。

2. 思考のプロセス（なぜ x+d に置くのか？）

まずはシンプルに、左から順番に扉を見ていくことを考えてみましょう。

例：d = 5、扉の位置が「10, 20, 40」の場合

まだ自販機は1台も置いていないので、最初の扉（位置 10）は必ず誰かがカバーしなければなりません。
では、自販機はどこに置くのが一番「得（最適）」でしょうか？

■ 扉10をカバーできる設置範囲

「5 〜 15」のどこに置いても、扉10をカバーできます。

  • 位置 10 に置くと ➔ カバー範囲は 5 〜 15 までしか届かない。
  • 位置 15 に置くと ➔ カバー範囲は 10 〜 20 まで届く！

右に置けば置くほど、これからチェックする予定の「さらに右側にある扉」を一緒に巻き込んでカバーできる可能性が上がります。
したがって、最初の扉の位置を x とすると、一番右の限界である x+d に置くのが最も得になります。

3 貪欲法（Greedy）の本質

この問題で初めて登場する重要なアルゴリズムが貪欲法（Greedy）です。
貪欲法とは、「その時点で一番良いと思われる選択を繰り返して、全体として最適な答えを目指す方法」です。

■ 身近な貪欲法の例

  • テストで、一番高い点数の問題から順番に解いていく。
  • トラックに、一番軽い荷物から順番に積めるだけ積む。
  • 今回の問題のように、一番右まで届くように自販機を置く。

このように「今できる一番得なこと」を愚直に選び続けるのが貪欲法の考え方です。しかし、いつでも貪欲法が正しい（最適解になる）わけではありません。
例えば、RPGゲームで「今持っているコインをその場で全て使う」のが後々有利になるとは限りませんよね。あとで強力な装備を買うために温存したほうが得な場合もあります。

➔ つまり、「今一番良さそう」を選んでも絶対に後悔しない（＝先々の選択の邪魔をしない）と証明できるときだけ、貪欲法が使えます。

■ では、この自販機問題では本当に大丈夫？

まず、まだカバーされていない一番左の扉に注目してみましょう。

10 20 40
▲
(まだ誰にもカバーされていない最初の扉)

この扉10は、この先どんなスマートな自販機の置き方をしようとも、絶対に誰かがカバーしなければならない扉です。「後回しにして、右側を先に考えてから決めよう」としても、この扉10が消えてくれるわけではありません。

【損をしない理由】

この扉10をカバーするために、どのみち自販機を最低1台消費することはここで確定しています。

それなら、その1台を 5 〜 15 の範囲のうち、一番右端である 15（x+d） に置いた方が絶対に損をしません。

これより左（10など）に置いてしまうと、右側をカバーできる能力がただ狭まるだけで、アルゴリズム上のメリットが何一つないからです。

そして、この1台を置いたあと、次に現れる「まだカバーされていない扉」に対しても、全く同じ論理が100%成り立ちます。
したがって、毎回「今できる一番得なこと（x+dへの配置）」を選び続ければ、最終的に全体でも最小台数（最適解）になると言えるのです。

貪欲法で考えるコツ

競技プログラミングで行き詰まったら、「端っこ（左や右）から順番に、その場しのぎで確定させられないか？」「今ここで一番極端な（得な）決定をしても、後悔しないかな？」と自問自答してみてください。それが貪欲法を見つける強力な足がかりになります。

4. アルゴリズムの全貌と実装

【処理手順】

  1. 一番左の扉から順番にスキャンする。
  2. 「最後に置いた自販機」でカバーできているなら何もしない（スルー）。
  3. カバーできていなければ、その扉の x+d の位置に新しく自販機を設置する。
  4. これを最後の扉まで繰り返す。

【計算量】

並んでいる扉のデータを左から右へちょうど1回だけ見ていくだけで全体の配置が確定します。

計算量: O(N)

■ 綺麗なC++実装例

変数 last_pos に「最後に置いた自販機の位置」を保存しておき、各扉との距離をチェックしながら更新していきます。 

#include <bits/stdc++.h>
using namespace std;

int main() {
    int N, d;
    while (cin >> N >> d) {
        if (N == 0 && d == 0) break;

        vector<int> doors(N);
        for (int i = 0; i < N; i++) {
            cin >> doors[i];
        }

        int ans = 0;
        // 最後に置いた自販機の位置（初期値は十分に左側のマイナス無限大）
        int last_pos = -2e9; 

        for (int i = 0; i < N; i++) {
            // 現在の扉が、最後に置いた自販機のカバー範囲（last_pos + d）を超えている場合
            if (doors[i] > last_pos + d) {
                ans++; // 新しく自販機を設置
                last_pos = doors[i] + d; // 最も右側の「扉の位置 + d」に設置する
            }
        }
        cout << ans << '\n';
    }
    return 0;
}
